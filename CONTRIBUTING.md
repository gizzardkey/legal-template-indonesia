# Contributing to Indonesia Legal Templates

First off, thank you for considering contributing to Indonesia Legal Templates! 🎉

This project aims to provide high-quality legal document templates for individuals, businesses, students, and developers in Indonesia.

We welcome contributions from everyone.

---

## How to Contribute

### 1. Fork the Repository

Fork this repository to your GitHub account and clone it locally.

```bash
git clone https://github.com/gizzardkey/legal-template-indonesia.git
cd indonesia-legal-templates
```

### 2. Create a Branch

Create a new branch for your changes.

```bash
git checkout -b feature/template-name
```

Example:

```bash
git checkout -b feature/nda-template
```

### 3. Make Your Changes

You may contribute by:

* Adding new legal templates
* Fixing typos or formatting issues
* Improving documentation
* Adding examples
* Updating outdated information
* Improving project structure

### 4. Commit Changes

Use clear commit messages.

```bash
git commit -m "Add NDA template"
```

### 5. Submit a Pull Request

Open a Pull Request and include:

* Description of changes
* Purpose of contribution
* Related issues (if any)

---

## Template Guidelines

All legal templates should:

* Be written in formal Indonesian language
* Use Markdown format (`.md`)
* Include clear titles
* Include party identification sections
* Include date and location fields
* Include signature sections when applicable
* Be easy to customize

---

## Directory Structure

Please place templates in the appropriate category.

```text
templates/
├── bisnis/
├── ketenagakerjaan/
├── legal/
└── umum/
```

---

## File Naming Convention

Use lowercase letters and hyphens.

Good examples:

```text
perjanjian-kerja-sama.md
surat-kuasa-khusus.md
kontrak-kerja-karyawan.md
```

Avoid:

```text
TemplateBaru.docx
SURATKUASA.MD
Template Final.doc
```

---

## Reporting Issues

If you find a problem:

1. Open an Issue
2. Describe the problem clearly
3. Provide steps to reproduce (if applicable)
4. Include screenshots or examples

---

## Pull Request Checklist

Before submitting a Pull Request, please ensure:

* [ ] Content has been reviewed
* [ ] Formatting follows repository standards
* [ ] File is placed in the correct directory
* [ ] File name follows naming conventions
* [ ] No duplicate template exists
